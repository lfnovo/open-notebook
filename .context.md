Open Notebook is an open-source, privacy-focused alternative to Google Notebook LM — an AI-powered research assistant with multi-modal content ingestion, semantic search, notebook-based organization, AI chat, and podcast generation.

## Summary

- README.md — project overview, features, quick-start links
- pyproject.toml — Python package definition and dependency declarations
- Makefile — common dev/build/release commands
- Dockerfile / Dockerfile.single — multi-service and single-container Docker images
- docker-compose.yml — local dev stack (SurrealDB + API + frontend)
- supervisord.conf / supervisord.single.conf — process supervision for single-container mode
- run_api.py — entrypoint to start the FastAPI server
- .env.example — reference for all required environment variables
- CONFIGURATION.md — full environment variable and model configuration reference
- CHANGELOG.md — release history
- open_notebook/ — core Python library: domain models, AI integration, LangGraph graphs, DB, utils
- api/ — FastAPI application: routers, services, auth, models
- frontend/ — Next.js 16 / React 19 web application
- commands/ — Surreal-Commands async job handlers (embedding, podcast, source)
- prompts/ — Jinja2 prompt templates for AI workflows
- tests/ — pytest test suite
- docs/ — user and developer documentation (Markdown)
- scripts/ — utility scripts (doc export, docker wait)
- examples/ — sample docker-compose configurations

## Constraints

- MUST start SurrealDB before the API; the API will fail on launch without a database connection.
- MUST run database migrations via the API startup (AsyncMigrationManager); never apply migrations manually.

## Guidance

- SHOULD use `uv` for Python dependency management (`uv run`, `uv add`).
- SHOULD consult `open_notebook/CLAUDE.md` for architectural decisions before making cross-cutting changes.
