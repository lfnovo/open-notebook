FastAPI router modules, one per resource domain. Each file defines path operations and delegates business logic to the corresponding service in `api/`.

## Summary

- auth.py — login / token endpoint
- notebooks.py — notebook CRUD
- sources.py — source creation, listing, deletion, reprocessing
- notes.py — note CRUD and listing per notebook
- chat.py — chat session creation, message sending, history retrieval
- source_chat.py — single-source chat endpoint
- commands.py — async job (command) status polling
- config.py — application config read endpoint
- context.py — RAG context retrieval endpoint
- credentials.py — API credential CRUD
- embedding.py — embedding trigger endpoint
- embedding_rebuild.py — full embedding rebuild endpoint
- episode_profiles.py — podcast episode profile CRUD
- insights.py — AI insight generation and retrieval
- languages.py — supported language listing
- models.py — AI model discovery and listing
- podcasts.py — podcast and episode CRUD and generation trigger
- search.py — semantic and full-text search endpoint
- settings.py — user settings CRUD
- speaker_profiles.py — podcast speaker profile CRUD
- transformations.py — transformation template CRUD and execution

## Guidance

- SHOULD keep each router to path operations only; no business logic inline.
