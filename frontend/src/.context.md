Next.js 16 / React 19 frontend source root. Provides the full user interface for Open Notebook, consuming the FastAPI backend via REST.

## Summary

- app/ — Next.js App Router pages and layouts
- app/(auth)/ — sign-in / sign-up pages
- app/(dashboard)/ — all authenticated pages (notebooks, sources, notes, chat, podcasts, search, settings)
- app/config/ — application configuration page
- app/layout.tsx — root layout (providers, global styles)
- app/globals.css — Tailwind CSS global stylesheet
- components/ — reusable React components organized by feature
- components/ui/ — Shadcn/ui primitive components
- components/layout/ — shell layout (sidebar, header, nav)
- components/notebooks/ — notebook list, detail, and creation components
- components/source/ — single-source view and chat components
- components/sources/ — source list and ingestion components
- components/podcasts/ — podcast and episode components
- components/search/ — search bar and results
- components/settings/ — settings forms
- components/auth/ — auth wrappers and guards
- components/common/ — generic shared components
- components/errors/ — error boundary and fallback components
- components/providers/ — React context providers (query client, theme, etc.)
- lib/ — API client, utility functions, store (Zustand), i18n helpers
- middleware.ts — Next.js middleware enforcing Clerk auth on protected routes
- test/ — Vitest unit tests for frontend utilities

## Constraints

- MUST add translation keys for all new user-visible strings (i18n-compatible).
- MUST NOT call FastAPI directly from components; use the API client in `lib/`.

## Guidance

- SHOULD use TanStack Query hooks for all server state (data fetching, mutations).
- SHOULD use Zustand for client-only UI state.
