# メンテナーガイド

## リリースプロセス

1. 変更を `main` ブランチにマージ
2. バージョン番号を更新
3. CHANGELOGを更新
4. タグを作成：`git tag v1.x.x`
5. タグをプッシュ：`git push --tags`

## コードレビュー

- 全PRは少なくとも1人のレビュアーによる承認が必要
- CIがグリーンであること
- ドキュメントの更新が含まれていること（該当する場合）

## セキュリティ

- 依存関係を定期的に監査
- 脆弱性レポートは `SECURITY.md` の手順に従う
