AI provider abstraction layer. Provides a unified interface over the Esperanto library to provision language, embedding, and speech models from 8+ providers (OpenAI, Anthropic, Google, Groq, Ollama, Mistral, DeepSeek, xAI).

## Summary

- models.py — `ModelManager` factory: selects and caches provider model instances; supports credential-based and env-var-based config
- provision.py — `provision_langchain_model()` helper used by all LangGraph graphs to obtain a model instance
- key_provider.py — retrieves API keys from credentials store or environment, with fallback logic
- model_discovery.py — dynamic discovery of available models per provider
- connection_tester.py — tests connectivity to AI provider endpoints

## Constraints

- MUST use `provision_langchain_model()` rather than instantiating provider clients directly in graph code.

## Guidance

- SHOULD add new providers through the Esperanto library interface; avoid vendor-specific SDK calls in this layer.
