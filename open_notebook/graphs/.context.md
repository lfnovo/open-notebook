LangGraph state machine workflows that power all AI-driven operations. Each file defines a compiled graph invoked by the API services.

## Summary

- source.py — content ingestion graph: extract text → chunk → embed → persist to SurrealDB
- chat.py — conversational chat graph with message history and RAG context injection
- ask.py — question-answering graph: semantic search → retrieve chunks → LLM synthesis
- source_chat.py — single-source focused chat graph (chat with one source directly)
- transformation.py — applies a user-defined transformation prompt to source content
- prompt.py — shared prompt assembly helpers for graph nodes
- tools.py — LangChain tools (search, retrieval) wired into graph nodes

## Constraints

- MUST use `provision_langchain_model()` for all model instantiation.
- MUST define a `StateDict` TypedDict for each graph's state.

## Guidance

- SHOULD keep node functions pure: receive state, return partial state update; avoid side effects inside nodes.
- SHOULD use SQLite checkpoint storage path `/data/sqlite-db/` for LangGraph persistence.
