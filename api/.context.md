FastAPI application layer. Exposes all REST endpoints consumed by the frontend, orchestrates service calls, and handles authentication and error mapping.

## Summary

- main.py — app factory: registers routers, CORS, global exception handlers, runs migrations on startup
- models.py — Pydantic request/response schemas (API-layer DTOs, separate from domain models)
- auth.py — simple password middleware (dev-only; replace with OAuth/JWT for production)
- client.py — internal async HTTP client for service-to-service calls
- routers/ — one router file per resource area (see routers/.context.md)
- notebook_service.py — business logic for notebook CRUD
- notes_service.py — business logic for note CRUD and search
- sources_service.py — business logic for source ingestion and management
- chat_service.py — delegates to `graphs/chat.py` for chat session handling
- command_service.py — tracks and retrieves async job (command) status
- context_service.py — assembles RAG context for chat requests
- credentials_service.py — CRUD for encrypted API credential records
- embedding_service.py — triggers embedding generation for sources/notes
- episode_profiles_service.py — podcast episode profile CRUD
- insights_service.py — AI-generated insights for notebooks
- models_service.py — provider model listing and validation
- podcast_service.py / podcast_api_service.py — podcast generation job submission and status
- search_service.py — semantic and full-text search across notebooks
- settings_service.py — user settings read/write
- transformations_service.py — applies transformation templates to sources

## Constraints

- MUST register every new router in `main.py`.
- MUST use `api/models.py` DTOs for all HTTP request/response bodies; never expose domain models directly.

## Guidance

- SHOULD keep routers thin: validate input, call a service, return response.
- SHOULD use the global exception handlers in `main.py` rather than catching `OpenNotebookError` in individual routers.
