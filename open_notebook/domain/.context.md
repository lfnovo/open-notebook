Pydantic v2 domain models and SurrealDB repository functions for all core Open Notebook entities. Every record type the application persists is defined here.

## Summary

- base.py — `OpenNotebookModel` base class with common fields (id, created, updated) and shared repository helpers
- notebook.py — `Notebook`, `Source`, `Note`, `ChatSession` models and their CRUD/search repository functions
- transformation.py — `Transformation` model (user-defined AI transformation templates)
- credential.py — `Credential` model for encrypted per-provider API key storage
- provider_config.py — `ProviderConfig` and `ModelConfig` for AI provider and model configuration records
- content_settings.py — `ContentSettings` for user-level content processing preferences

## Constraints

- MUST use Pydantic v2 field declarations and validators.
- MUST go through repository functions for all DB access; never write raw SurrealQL outside `database/` or `domain/`.

## Guidance

- SHOULD keep domain models free of HTTP/FastAPI concerns; request/response schemas belong in `api/models.py`.
