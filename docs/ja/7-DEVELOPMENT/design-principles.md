# デザイン原則

## コア原則

1. **プライバシー最優先**: ユーザーデータはユーザーの管理下に
2. **プロバイダー中立**: 単一のAIプロバイダーに依存しない
3. **シンプルさ**: 複雑さより明快さを優先
4. **拡張性**: オープンソースとして誰でも拡張可能
5. **オフライン対応**: 可能な限りローカル実行をサポート

## アーキテクチャ原則

- **関心の分離**: フロントエンド・API・DBの明確な分離
- **非同期ファースト**: 処理は非同期で
- **設定より規約**: デフォルトで動作し、必要に応じて設定可能
